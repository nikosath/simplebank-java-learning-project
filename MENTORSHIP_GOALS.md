# Mentorship Goals (Flexible Pace)

This note is a companion to [README.md](README.md). The README is the source of truth for version requirements, workflow, schedule, and acceptance criteria.

## Mentee Goals
- Aim to complete the README core requirements for each version, with the main focus on understanding and correctness.
- Keep a basic automated test suite in place for the critical flows described in the README.
- Use the README’s Mentor and Mentee Workflow, then share the PR link and tag name when a version is ready.
- Treat the target months in the README as guideposts rather than hard deadlines.

## Mentor Goals
- Review work against the README core requirements and help clear blockers.
- Focus feedback on correctness, learning, and major risks before polish.
- Respond in a reasonable timeframe when practical, and adjust pacing if the plan needs to shift.

## Stretch Goals
- Optional and only after the core README requirements for that version are complete.
- Keep stretch work small and time-boxed.
